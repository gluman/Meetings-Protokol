        🔧 API и MCP
        

          
REST API (требуется Authorization: Bearer <key>):

          
    
            
  • POST /api/v1/transcribe — multipart: file, prompt
    • 
            
  • GET /api/v1/jobs/{id} — статус
    • 
            
  • GET /api/v1/download/{filename} — DOCX
    • 
            
  • GET /api/v1/jobs — список завершённых
    • 
          

          
MCP (Model Context Protocol):

          
    
            
  • GET /mcp/info — discovery
    • 
            
  • GET /mcp/sse — SSE transport
    • 
            
  • POST /mcp/rpc — JSON-RPC 2.0
    • 
            
  • Инструменты: transcribe_meeting, get_protocol, list_protocols
